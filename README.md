# TCM AI Agent

一个基于 **本地大模型（Qwen3-32B）+ RAG + 结构化中医知识库 + 推理链 + 用户状态管理** 的中医健康咨询系统。

---

## 🚀 项目亮点

### 1️⃣ 结构化中医知识库

- 构建“症状 → 证型 → 方剂”知识体系  
- 引入：关键症状 / 舌象 / 脉象 / 排除项  
- 显著提升检索精度与可解释性  

---

### 2️⃣ 推理链（核心创新）

不同于传统 RAG 的“检索 → 生成”，本系统采用：

```text
用户输入
→ 症状抽取
→ 证型打分
→ Top-K排序
→ RAG检索证据
→ 大模型生成解释
```

实现类中医辨证的推理流程。

---

### 3️⃣ 用户状态与历史记忆（重要升级）

系统为每个用户维护：

- 当前健康状态（证型 + 置信度）
- 历史记录（多次输入）
- 状态变化轨迹（证型演变）

```text
风寒表证 → 风热表证 → 恢复期
```

使系统具备：

- 连续分析能力  
- 状态演化理解能力  
- 更接近真实临床逻辑  

---

### 4️⃣ 本地大模型（隐私 & 成本友好）

- 基于 vLLM 部署 Qwen3-32B  
- OpenAI 兼容接口  
- 支持离线运行  

---

### 5️⃣ 医疗安全机制

- 危险症状检测（如胸痛、呼吸困难等）  
- 自动提示线下就医  
- 降低误导风险

## 🧠 系统架构

```text
            ┌──────────────┐
            │ 用户输入症状 │
            └──────┬───────┘
                   ↓
        ┌────────────────────┐
        │ 症状抽取（规则）   │
        └──────┬─────────────┘
               ↓
        ┌────────────────────┐
        │ 证型打分排序       │
        │ (rule-based)       │
        └──────┬─────────────┘
               ↓
        ┌────────────────────┐
        │ 用户状态更新       │
        │ (history + state)  │
        └──────┬─────────────┘
               ↓
        ┌────────────────────┐
        │ RAG 检索           │
        └──────┬─────────────┘
               ↓
        ┌────────────────────┐
        │ Qwen3-32B 生成     │
        └──────┬─────────────┘
               ↓
        ┌────────────────────┐
        │ 结构化输出         │
        └────────────────────┘
```

## ⚙️ 安装与运行

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

```bash
cp .env.example .env
```

## 🤖 启动本地模型（vLLM）

```bash
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen3-32B \
  --served-model-name qwen3-32b \
  --host 0.0.0.0 --port 8000
```

## 📊 示例流程

输入：

```text
恶寒重、发热轻、无汗、头痛、鼻塞
```

系统输出：

- 证型排序：风寒表证 > 风热表证  
- 推荐治法：辛温解表  
- 推荐方剂：麻黄汤  

再次输入：

```text
发热、咽痛、黄痰
```

系统识别：

```text
风寒表证 → 风热表证（证型演变）
```
